# Ticket 3b.7: Create migration tool (SQLite → Firestore)

## Summary
Create a CLI tool to migrate existing SQLite learnings to Firestore cloud.

## Context
Users need a way to migrate their local SQLite data to Firestore when switching to cloud mode.

## Acceptance Criteria
- [ ] `echo admin migrate --from sqlite --to firestore` command
- [ ] Reads all learnings from SQLite
- [ ] Generates embeddings for learnings without vectors
- [ ] Batch-writes to Firestore
- [ ] Verifies count match after migration
- [ ] Reports summary (total, migrated, failed)
- [ ] Dry-run mode (`--dry-run`)
- [ ] Resume capability (skips already-migrated)

## Technical Details
```bash
echo admin migrate --from sqlite --to firestore \
  --firestore-project-id=my-gcp-project \
  --firestore-collection=echo_learnings \
  --embedder=local \
  --model-path=/path/to/model.onnx
```

Flow:
1. Read all learnings from SQLite
2. For each learning without embedding, generate via embedder
3. Batch-write to Firestore (500 docs per batch)
4. Verify count match
5. Report summary
