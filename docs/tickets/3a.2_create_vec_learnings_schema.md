# Ticket 3a.2: Create vec_learnings virtual table + triggers

## Summary
Create the sqlite-vec virtual table schema and triggers to keep it in sync with learnings_data.

## Context
We need a vec0 virtual table to store embeddings and metadata. Triggers will ensure the vector table stays in sync with the main learnings_data table.

## Acceptance Criteria
- [ ] `vec_learnings` virtual table created on first run
- [ ] Triggers for INSERT, UPDATE, DELETE on learnings_data
- [ ] Table schema matches Tech Spec (384 dims, metadata columns)
- [ ] Migration runs automatically on startup

## Technical Details
```sql
CREATE VIRTUAL TABLE IF NOT EXISTS vec_learnings USING vec0(
    embedding float[384],
    learning_id TEXT,
    project TEXT,
    scope TEXT,
    type TEXT,
    tags TEXT,
    question TEXT,
    answer TEXT
);
```

Triggers:
- `vec_learnings_insert`: After INSERT on learnings_data, insert into vec_learnings
- `vec_learnings_update`: After UPDATE on learnings_data, update vec_learnings
- `vec_learnings_delete`: After DELETE on learnings_data, delete from vec_learnings
