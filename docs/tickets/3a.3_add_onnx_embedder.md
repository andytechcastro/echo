# Ticket 3a.3: Add ONNX runtime dependency + embedder implementation

## Summary
Implement the ONNX-based local embedder using all-MiniLM-L6-v2 model.

## Context
We need a local embedding implementation that runs the all-MiniLM-L6-v2 model via ONNX Runtime. This replaces external API calls for Phase 3a.

## Acceptance Criteria
- [ ] `github.com/yalue/onnxruntime_go` added to go.mod
- [ ] `ONNXEmbedder` struct implements `domain.Embedder` interface
- [ ] `Embed()` returns float32[384] vector
- [ ] `Dimensions()` returns 384
- [ ] `Provider()` returns "local-onnx"
- [ ] Tokenization handles English text correctly
- [ ] Mean pooling + normalization implemented
- [ ] Unit tests for embedding (mock or real model)

## Technical Details
- Model: all-MiniLM-L6-v2 (384 dims, ~22MB)
- Runtime: `github.com/yalue/onnxruntime_go` (CGO-based)
- Tokenizer: Simple BPE or WordPiece (can use `github.com/dlclark/regexp2` for basic tokenization)
- Inference flow: tokenize → run ONNX → mean pooling → normalize → return
