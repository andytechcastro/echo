# Ticket 3a.5: Implement vector save (populate vec_learnings on Save)

## Summary
Update the SaveLearning usecase to generate embeddings and save them to vec_learnings.

## Context
When a learning is saved, we need to generate an embedding (if embedder is available) and store it in the vec_learnings table alongside the learning data.

## Acceptance Criteria
- [ ] SaveLearning generates embedding when embedder is not nil
- [ ] Embedding text = question + " " + answer
- [ ] vec_learnings row created with learning_id and metadata
- [ ] If embedding fails, save still succeeds (warning logged)
- [ ] Existing save behavior unchanged when embedder is nil

## Technical Details
```go
func (s *SaveLearning) Execute(ctx context.Context, input *domain.SaveInput) (string, error) {
    learning := domain.NewLearning(...)

    // Generate embedding if available
    if s.embedder != nil {
        textToEmbed := input.Question + " " + input.Answer
        embedding, err := s.embedder.Embed(ctx, textToEmbed)
        if err != nil {
            slog.Warn("embedding failed, saving without vector", "error", err)
        } else {
            learning.Embedding = embedding
        }
    }

    id, err := s.store.Save(ctx, learning)
    if err != nil {
        return "", err
    }

    // Save vector separately (sqlite-vec doesn't support transactions with FTS5)
    if learning.Embedding != nil {
        err := s.store.SaveVector(ctx, id, learning.Embedding, map[string]string{
            "project":  learning.Project,
            "scope":    string(learning.Scope),
            "type":     string(learning.Type),
            "tags":     strings.Join(learning.Tags, ","),
            "question": learning.Question,
            "answer":   learning.Answer,
        })
        if err != nil {
            slog.Warn("vector save failed, learning saved without vector", "error", err)
        }
    }

    return id, nil
}
```
