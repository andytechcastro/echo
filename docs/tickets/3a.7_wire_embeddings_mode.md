# Ticket 3a.7: Add --mode embeddings wiring in main.go

## Summary
Wire up the `--mode embeddings` flag in main.go to initialize the ONNX embedder and vector store.

## Context
The `--mode embeddings` flag exists but is a no-op. We need to wire it to actually initialize the ONNX embedder and vector store.

## Acceptance Criteria
- [ ] `--mode embeddings` initializes ONNX embedder
- [ ] `--mode embeddings` creates SQLiteVecStore alongside SQLiteFTS5Store
- [ ] `--embedder` flag supports "local" value
- [ ] `--model-path` flag added for model location override
- [ ] Existing `--mode local` behavior unchanged
- [ ] Error message if model not found and download fails

## Technical Details
Update `runServe()` in main.go:

```go
var embedderImpl domain.Embedder

switch cfg.Mode {
case "local":
    // No embedder, BM25 only
case "embeddings":
    // Initialize ONNX embedder
    embedderImpl, err = embedder.NewONNXEmbedder(cfg.ModelPath)
    if err != nil {
        return fmt.Errorf("create embedder: %w", err)
    }
    logger.Info("local embedder initialized", "model", cfg.ModelPath, "dims", embedderImpl.Dimensions())
case "cloud":
    return fmt.Errorf("cloud mode not yet implemented (Phase 3b)")
}

// Pass embedder to usecases
saveUC := usecase.NewSaveLearning(textStore, embedderImpl, projDet, identDet)
searchUC := usecase.NewSearchLearning(textStore, projDet, embedderImpl)
```

Config additions:
```go
type Config struct {
    // ... existing
    EmbedderType string // "local", "vertex-ai", "openai", "cohere"
    ModelPath    string // path to ONNX model
}
```

CLI flags:
```bash
echo serve --mode embeddings --embedder local --model-path /path/to/model.onnx
```
