# Ticket 3b.4: Implement FirestoreStore (Save, Search, kNN)

## Summary
Implement Firestore as a cloud TextStore backend with native kNN vector search.

## Context
Firestore now supports native vector search (kNN) with `vector<float32>` field type. This enables real-time shared memory across machines.

## Acceptance Criteria
- [ ] `FirestoreStore` implements `domain.TextStore`
- [ ] Save creates document with embedding vector field
- [ ] Search uses Firestore kNN query
- [ ] Project filtering applied
- [ ] Limit respected
- [ ] Auth via Google Cloud IAM
- [ ] Integration tests with Firestore emulator

## Technical Details
- Collection: `echo_learnings`
- Vector field: `embedding` (vector<float32>(768))
- Index: COSINE_SIMILARITY on embedding field
- Auth: Google ADC (service account)
- Region: Configurable (default: us-central1)

```go
type FirestoreStore struct {
    client     *firestore.Client
    projectID  string
    collection string
    embedder   domain.Embedder
}
```
