# Ticket 3b.5: Implement HybridStore (cloud + local fallback)

## Summary
Implement HybridStore that tries cloud first and falls back to local SQLite on failure.

## Context
For production resilience, we need a store that uses cloud when available but gracefully falls back to local BM25/vector search when cloud is unreachable.

## Acceptance Criteria
- [ ] `HybridStore` implements `domain.TextStore`
- [ ] Save writes to both cloud and local
- [ ] Search tries cloud first, falls back to local
- [ ] GetByID tries cloud first, falls back to local
- [ ] Delete removes from both cloud and local
- [ ] Logging of fallback events
- [ ] Unit tests for fallback scenarios

## Technical Details
```go
type HybridStore struct {
    cloud  *FirestoreStore
    local  *SQLiteFTS5Store
    logger *slog.Logger
}

func (h *HybridStore) Search(ctx context.Context, query *domain.SearchQuery) ([]domain.SearchResult, error) {
    // Try cloud first
    results, err := h.cloud.Search(ctx, query)
    if err == nil && len(results) > 0 {
        return results, nil
    }

    // Fallback to local
    h.logger.Warn("cloud search failed, falling back to local", "error", err)
    return h.local.Search(ctx, query)
}
```
